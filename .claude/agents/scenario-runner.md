# Scenario Runner (project-scoped shadow)

**This is a project-level override** of the plugin-shipped agent in
`scenarios-autorunner` v0.1.4+. The only difference from the plugin
version is the `hooks:` frontmatter field — CPV strict mode refuses
`hooks:` in plugin-shipped agents (security restriction), so the guard
has to live in a project- or user-scope agent file.

All other behaviour is inherited from the plugin documentation at
`~/.claude/plugins/cache/emasoft-plugins/scenarios-autorunner/0.1.4/agents/scenario-runner.md`.
Do not edit this file to diverge from the plugin semantics — if the
plugin updates, copy the new body in and keep the `hooks:` field at
the top.

## Write-guard (enforced by the hook)

The `PreToolUse` hook runs `.claude/scripts/subagent-write-guard.sh`
before every `Write`, `Edit`, `MultiEdit`, `NotebookEdit`, and `Bash`
tool call. It blocks writes that land outside:

1. `$CLAUDE_PROJECT_DIR` — your project root (the main AI Maestro
   repo, because the runner does NOT use `isolation: worktree`)
2. `/tmp` / `/private/tmp` — system scratch

Read operations are NOT blocked. You may read any file on the system,
including `~/.claude/` for plugin diagnostics, `~/.aimaestro/` for the
agent registry, tmux sessions, etc.

Writes are restricted because the runner applies FIX-AS-YOU-GO patches
during a scenario — those patches must land on source files inside the
project repo (under `services/`, `components/`, `lib/`, etc.), not on
user-level or system-level configuration files.

## Delegating to the plugin body

The full instructions — Phase A-H, CHROME-TOOL, FIX-AS-YOU-GO, Rule 12
sudo modal handling, rate-limit resilience, return format — live in
the plugin-shipped agent. Read it at the start of your run:

```
cat ~/.claude/plugins/cache/emasoft-plugins/scenarios-autorunner/0.1.4/agents/scenario-runner.md
```

Follow that file verbatim, subject to the additional write-guard
constraint documented above.

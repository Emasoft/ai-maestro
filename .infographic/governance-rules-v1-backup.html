Closed Team Rules

  • All teams are closed after governance simplification
  • Internal messaging: any member → any member (free)
  • External messaging: must route through COS gateway
  • COS manages roster: add/remove agents
  • Manager creates teams, assigns COS
  • Each agent can belong to one team only
  • Groups replace old “open teams” (no governance)
  • Team deletion requires governance password

6 — SECURITY MEASURES

Authentication

  • Phase 1: localhost-only, no auth required
  • Bearer token auth for agent-to-agent API calls
  • API keys: 256-bit random, SHA-256 hashed at rest
  • Constant-time key validation (timing-safe)
  • MANAGER + COS titles restricted to Claude clients

Rate Limiting

  • All password-protected endpoints rate-limited
  • Atomic check-and-record (no TOCTOU race)
  • Per-agent rate limit keys
  • Reset on successful auth
  • AMP route: max requests/min per agent

Cryptography

  • Ed25519 host keypairs (0o600 file perms)
  • Signed attestations with timestamp freshness (5 min)
  • Recipient binding prevents replay across hosts
  • AMP messages signed with Ed25519 per-agent keys
  • bcrypt for governance passwords (72-byte limit)

Transfer Rules

  • Only MANAGER and COS can initiate transfers
  • Cross-host requests require dual approval
  • Incoming requests always forced to pending (prevents pre-approved injection)
  • Old role-plugin uninstalled before new one installed
  • Transfer history preserved in registry

Governance Password

  • Required for: team deletion, manager assignment, COS assignment
  • bcrypt hashed (cost factor 10)
  • Rate-limited: lockout after repeated failures
  • System owner (web UI) can override with password
  • 72-character max (bcrypt byte limit)
8 — GROUPS (LIGHTWEIGHT COLLECTIONS)

What Groups Are

  • Lightweight agent collections for broadcast messaging
  • Replace old “open teams” concept
  • No governance, no COS, no kanban
  • Just a subscriber list
  • Agents can subscribe to multiple groups

Group Operations

  • Create / update / delete groups
  • Subscribe / unsubscribe agents
  • Broadcast notify all subscribers
  • File-based storage: ~/.aimaestro/teams/groups.json
  • API: GET/POST /api/groups

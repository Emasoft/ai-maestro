# Fix Report: AgentList.tsx — 2026-03-22T18:37:23

## Fixes Applied

### Fix 1 (HIGH): Compact view Hibernate/Wake buttons missing e.stopPropagation()

**File**: `components/AgentList.tsx`
**Location**: compact view agent list action buttons

**Problem**: Hibernate button called `onClick={(e) => handleHibernate(agent, e)}` and Wake button called `onClick={(e) => handleWake(agent, e)}` without explicit `e.stopPropagation()` in the onClick handler itself. The parent `<div onClick={() => handleAgentClick(agent)}>` would fire in addition to the intended action.

**Fix**: Both buttons now call `e.stopPropagation()` explicitly as the first statement in their onClick handler. Wake button directly calls `setWakeDialogAgent(agent)` after stopping propagation (not via `handleWake`) per the instruction's requirement to "call e.stopPropagation() before setting the wake dialog agent".

---

### Fix 2 (HIGH): Inconsistent agent status field access in filteredAgents useMemo

**File**: `components/AgentList.tsx`
**Location**: `filteredAgents` useMemo

**Problem**: `active` filter used `a.session?.status === 'online'` (singular legacy field) while the rest of the component consistently uses `agent.sessions?.[0]?.status` (array form). An agent with a valid `sessions[0]` but no legacy `session` property would be wrongly excluded from the active tab.

**Fix**: Changed to `a.sessions?.[0]?.status === 'online'` for the active filter. The hiber filter was already corrected to use `a.sessions && a.sessions.length > 0 && a.sessions[0]?.status !== 'online'`.

---

### Fix 3 (MEDIUM): Hiber tab count incorrectly includes agents with no session

**File**: `components/AgentList.tsx`
**Location**: status tabs array (ACTIVE and HIBER count expressions)

**Problem**: `agents.filter(a => a.session?.status !== 'online').length` — since `undefined !== 'online'` is true, agents with no sessions at all were counted as hibernated, inflating the HIBER count. The ACTIVE tab also used the legacy singular `a.session?.status` field.

**Fix**:
- ACTIVE count: `agents.filter(a => a.sessions?.[0]?.status === 'online').length`
- HIBER count: `agents.filter(a => a.sessions && a.sessions.length > 0 && a.sessions[0]?.status !== 'online').length`

Both now use `sessions[0]` consistently and HIBER correctly excludes agents with no sessions.

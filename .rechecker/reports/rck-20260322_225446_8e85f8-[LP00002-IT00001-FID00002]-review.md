### BUG: Catch block handles all errors as "Invalid request body"
**Severity**: medium
**Location**: `POST` function, `catch` block
**Problem**: The `catch` block in the `POST` function is too broad. It catches any error that occurs during the execution of the `try` block and always returns a 400 status with the message "Invalid request body". This is problematic because errors other than invalid JSON parsing (e.g., network issues, errors from `buildPlugin` that are not caught by its internal error handling, or unexpected runtime errors) will also result in the same generic error message, making debugging difficult and providing misleading information to the client.
**Fix**: Differentiate between errors. Specifically, if `request.json()` fails, it's likely a malformed JSON, which warrants a 400. Other errors should ideally be logged and perhaps returned with a 500 status code, or at least a more generic "Internal Server Error" message, unless specific error types are being caught and handled. For this specific case, since `buildPlugin` is expected to handle its own errors and return them in `result.error`, the `catch` block should primarily focus on errors from `request.json()`.

  let body: any;
  try {
    body = await request.json();
  } catch (error) {
    // This catch block specifically handles errors during JSON parsing
    return NextResponse.json(
      { error: 'Invalid request body' },
      { status: 400 }
    );
  }

  try {
    const result = await buildPlugin(body);

    if (result.error) {

  }
}

# Loop 2 Correctness Review — SkillPicker.tsx

### BUG: `filteredMarketplaceSkills` filter crashes on undefined description
**Severity**: high
**Location**: `filteredMarketplaceSkills` useMemo — filter predicate `s.description.toLowerCase()`
**Problem**: `s.description` may be undefined/null for marketplace skills. Calling `.toLowerCase()` on undefined throws a runtime TypeError.
**Fix**: Replace `s.description.toLowerCase()` with `(s.description ?? '').toLowerCase()`. Similarly for `s.name`: replace `s.name.toLowerCase()` with `(s.name ?? '').toLowerCase()`.

### BUG: Duplicate React keys in marketplace skills list
**Severity**: high
**Location**: `filteredMarketplaceSkills.map(skill => (<div key={skill.id} ...))` in the `activeTab === 'marketplace'` render block
**Problem**: `key={skill.id}` uses only the skill id, but marketplace skills from different sources can share the same id. This causes duplicate keys, React reconciliation warnings, and incorrect rendering.
**Fix**: Change to `key={key}` where `key` is the compound key already computed as `` `marketplace:${skill.marketplace ?? 'unknown'}:${skill.id}` `` (which is the same value used for `selectedKeys.has(key)` and `onRemoveSkill(key)`).

### BUG: `getSkillKey` marketplace case uses wrong separator format
**Severity**: medium
**Location**: `getSkillKey` function — `case 'marketplace'` branch
**Problem**: The key format must be `marketplace:${skill.marketplace}:${skill.id}` to be consistent with how keys are looked up in `selectedKeys`. If the format differs from how keys are generated elsewhere, selection state breaks.
**Fix**: Ensure the marketplace case returns `` `marketplace:${skill.marketplace ?? 'unknown'}:${skill.id}` ``.

### BUG: Unguarded `skill.name` render in marketplace tab
**Severity**: medium
**Location**: Marketplace skills map JSX — `<p className="text-sm font-medium text-gray-200">{skill.name}</p>`
**Problem**: `skill.name` can be null/undefined. Renders as blank, inconsistent with aria-label which guards it.
**Fix**: Replace `{skill.name}` with `{skill.name ?? 'Unnamed'}`.

### BUG: Unguarded fields in marketplace skill description fallback
**Severity**: medium
**Location**: Marketplace skills map JSX — `<p className="text-xs text-gray-500 truncate">{skill.description || \`${skill.plugin} / ${skill.marketplace}\`}</p>`
**Problem**: Fallback string interpolation uses `skill.plugin` and `skill.marketplace` without null guards, rendering "undefined / undefined" when either is missing.
**Fix**: Replace with `{skill.description ?? \`${skill.plugin ?? 'Unknown'} / ${skill.marketplace ?? 'unknown'}\`}`.

### BUG: Missing null guards in marketplace skill aria-label
**Severity**: low
**Location**: Marketplace skills map JSX — `aria-label={...}` attribute
**Problem**: If `skill.name`, `skill.plugin`, or `skill.marketplace` are null/undefined, the aria-label string contains "null" or "undefined", breaking accessibility.
**Fix**: Use: `` aria-label={`${skill.name ?? 'Unnamed'}: ${skill.description ?? `${skill.plugin ?? 'Unknown'} / ${skill.marketplace ?? 'unknown'}`}`} ``

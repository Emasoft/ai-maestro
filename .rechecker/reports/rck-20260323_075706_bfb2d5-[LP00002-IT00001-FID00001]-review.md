### BUG: Incorrect `diff_count` calculation for dry-run
**Severity**: medium
**Location**: `DRY_RUN` block
**Problem**: In the dry-run block, the `diff_count` is calculated using `git diff --stat "$CURRENT_BRANCH...$branch"`. The output of `git diff --stat` includes a summary line at the end (e.g., "3 files changed, 10 insertions(+), 5 deletions(-)"). The current logic subtracts 1 from `diff_count` to account for this summary line, assuming it's always present. However, if there are no changes between the branches, `git diff --stat` will produce no output, and `wc -l` will return 0. In this case, `diff_count -le 1` will incorrectly evaluate to `0 -le 1`, leading to the message "no changes — would skip" even if there are no changes. More critically, if `diff_count` is 0, then `diff_count - 1` will be -1, which is then used in the output string `($((diff_count - 1)) files changed)`. This will display a negative number of files changed, which is misleading.
**Fix**: The condition `[[ "$diff_count" -le 1 ]]` should be `[[ "$diff_count" -eq 0 ]]` to correctly identify branches with no changes. The number of files changed should be `diff_count` if `diff_count` is greater than 0, otherwise 0. A more robust way to count changed files is to parse the `git diff --stat` output or use a different `git diff` option that directly gives the count. For simplicity, we can adjust the condition and the displayed count.

```bash
# DRY_RUN block
if $DRY_RUN; then
  echo ""
  echo "DRY RUN — would merge these branches:"
  echo "$BRANCHES" | while read -r branch; do
    # Calculate diff_count more robustly
    diff_output=$(git diff --stat "$CURRENT_BRANCH...$branch" --ignore-submodules 2>/dev/null)
    diff_count=$(echo "$diff_output" | wc -l | tr -d ' ')

    if [[ -z "$diff_output" ]]; then # No output means no changes
      echo "  $branch  (no changes — would skip)"
    else
      # Subtract 1 for the summary line if it exists
      actual_files_changed=$((diff_count - 1))
      if [[ "$actual_files_changed" -lt 0 ]]; then # Handle case where only summary line exists or no changes
        actual_files_changed=0
      fi
      echo "  $branch  ($actual_files_changed files changed)"
    fi
  done
  exit 0
fi
```

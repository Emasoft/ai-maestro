# Fix Report: FID00011 — Missing onRemoveSkill in RepoScanner

## Summary
Fixed the root cause of the bug: `RepoScanner` never declared or accepted `onRemoveSkill`, so skills added from a repo could never be deselected.

## Root Cause
`RepoScannerProps` lacked the `onRemoveSkill` prop entirely. The component also disabled the add button when a skill was already selected (instead of toggling removal), making deselection impossible. `SkillPicker` was also passing an undeclared `getSkillKey` prop to `RepoScanner` which was silently ignored.

## Files Changed

### components/plugin-builder/RepoScanner.tsx
- Added `onRemoveSkill: (key: string) => void` to `RepoScannerProps` interface.
- Added `onRemoveSkill` to the component destructured parameters.
- Added `Check` icon import (used for selected state visual feedback).
- Changed the per-skill button from `disabled={isSelected}` + always showing `Plus` to a true toggle: calls `onRemoveSkill(key)` when selected, `handleAddSkill(skill)` when not; shows `Check` icon when selected, `Plus` when not.

### components/plugin-builder/SkillPicker.tsx
- Added `onRemoveSkill={onRemoveSkill}` to the `<RepoScanner>` JSX usage.
- Removed the unused `getSkillKey={getSkillKey}` prop (was never declared in `RepoScannerProps` and silently ignored; the key is computed correctly inline within `RepoScanner`).

## Verification
- `RepoScannerProps` now fully matches the props passed from `SkillPicker`.
- The toggle logic mirrors the pattern used for Core and Marketplace skill rows in `SkillPicker`.
- No other callers of `RepoScanner` exist in the codebase (only used in `SkillPicker.tsx`).

# Fix Report: FID00017 — remote-install.sh error suppression

**File:** `scripts/remote-install.sh`
**Finding file:** `rck-20260322_195909_57da28-[LP00002-IT00002-FID00017]-review.md`

## Changes Applied

### 1. git stash pop — removed `2>/dev/null` (line 951)
- **Before:** `if ! git stash pop --quiet 2>/dev/null;`
- **After:** `if ! git stash pop --quiet;`
- Conflict messages from git are now visible to the user.

### 2. npm install for gateways (update path) — removed `2>/dev/null` and `|| true` (line 984)
- **Before:** `(cd "${gw}-gateway" && npm install --silent 2>/dev/null || npm install 2>/dev/null || true)`
- **After:** `(cd "${gw}-gateway" && npm install || maestro_warn "npm install failed for ${gw}-gateway — skipping")`
- Failures are now visible; the `maestro_warn` fallback is preserved and correctly triggered.

### 3. npm install for gateways (fresh install path) — removed silent first attempt (lines 1037-1041)
- **Before:** `npm install --silent 2>/dev/null || npm install || { maestro_warn ...; continue; }`
- **After:** `npm install || { maestro_warn ...; continue; }`
- Single visible call; the `maestro_warn`+continue fallback is preserved.

### 4. pm2 start commands — removed `2>/dev/null` (lines 1127-1133)
- All three `pm2 start` invocations in the "Start the service" block had `2>/dev/null` removed.
- `pm2 save 2>/dev/null || true` was intentionally left untouched (non-critical, as instructed).

### 5. _install_npm_global — removed `2>/dev/null` from interactive branch (lines 754-757)
- The NON_INTERACTIVE vs interactive branching was the sole difference between the two paths (one suppressed stderr, the other did not). Since the only difference was the suppression, the branches were collapsed into a single `npm install -g "$pkg"` call, eliminating the dead branch.

### 6. yarn install (both paths) — simplified to single call (lines 962, 1018)
- **Before (both occurrences):** `yarn install --silent 2>/dev/null || yarn install || maestro_warn "..."`
- **After (both occurrences):** `yarn install || maestro_warn "yarn install had errors — continuing"`
- The silent-first-then-verbose pattern was suppressing errors on the first attempt. Collapsed to a single visible call; the `maestro_warn` fallback is preserved.

## Intentionally Not Changed
- `pm2 save 2>/dev/null || true` — non-critical operation, intentional per instructions.
- All `2>/dev/null` in `open_browser` calls — those suppress GUI tool errors, not installation errors.
- `2>/dev/null` in `uninstall` — those are cleanup operations on potentially-absent resources, where silence is correct.
- `git pull origin main 2>/dev/null || git pull` — the fallback `git pull` surfaces errors if both fail.

Passing a synchronous `ServiceResult<T>` value to `sendServiceResult(res, value)` is **correct**. Adding `await` to a synchronous call is harmless but unnecessary, and not a bug.

All functions that ARE async (return `Promise<ServiceResult<T>>`) were already correctly called with `await` in the original file. Verified via grep of `sendServiceResult(res, await ...)`.

### BUG 2: "Inconsistent error handling for sendServiceResult" — NOT A BUG

The existing `sendServiceResult` implementation correctly handles the `error` vs `data` branching. The service layer contract is consistent: a result object has either `.error` (string) or `.data` (payload), never both. No change needed.

### BUG 3: "Inconsistent await usage in handle method" — NOT A BUG

`sendJson` is a synchronous function (`res.writeHead` + `res.end` are synchronous Node.js calls). Adding `await` to a synchronous function is harmless but unnecessary. Not a real bug.

## Changes Applied

None. The original file is correct. All false-positive `await` additions that were speculatively applied during analysis were reverted before finalizing.

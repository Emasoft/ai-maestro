# Fix Report: FID00019 — plugin-builder-service.ts

**File**: `services/plugin-builder-service.ts`
**Date**: 2026-03-22

## Fix 1: Double activeOps Decrement in `buildPlugin` (HIGH)

### Root Cause
The original code had `.finally(() => { activeOps = Math.max(0, activeOps - 1) })` chained on the `runBuild(...)` fire-and-forget promise, AND a `catch` block that also decremented `activeOps` via `opStarted`. While these two paths are mutually exclusive in the original code (sync errors hit the `catch` before `runBuild` is reached; the `.finally()` fires on async completion), the design was fragile: there was no `finally` block on the outer `try`, meaning `activeOps` was NOT decremented on the success path (202 return) — only on async `runBuild` completion or sync error. The instructions required eliminating the `.finally()` on `runBuild` and relying solely on the outer block.

### Fix Applied
1. Removed `.finally(() => { activeOps = Math.max(0, activeOps - 1) })` from the `runBuild(...)` promise chain.
2. Converted the outer `catch`-only block into a `try/catch/finally` structure:
   - `catch`: handles logging and returns the 500 error response (no decrement here).
   - `finally`: decrements `activeOps` exactly once using the `opStarted` guard, covering both the success path (202) and the sync-error path (500).

This makes it impossible for `activeOps` to be decremented twice for a single operation, and guarantees it is always decremented exactly once when `buildPlugin` returns.

### Lines Changed
- Lines 322–406 (outer `try/catch` restructured to `try/catch/finally`)
- Lines 379–393 (`.finally()` removed from `runBuild` call)

## Fix 2: Path Traversal in `generateManifest` (HIGH)

### Root Cause
`installPath` was built with `path.join(os.homedir(), '.claude', 'plugins', 'marketplaces', group.marketplace)`. If `group.marketplace` contained `../` sequences (e.g., `../../../etc`), `path.join` would resolve them and the resulting path could escape the intended `.claude/plugins/marketplaces/` directory. `path.join` does NOT strip traversal sequences — it normalizes them.

### Fix Applied
Added `path.basename(group.marketplace)` before the `path.join` call. `path.basename` returns only the final path component, stripping all directory separators and traversal sequences, so any value like `../../etc/passwd` becomes just `passwd`, and a valid name like `my-marketplace` is unchanged.

```typescript
const sanitizedMarketplace = path.basename(group.marketplace)
const installPath = path.join(os.homedir(), '.claude', 'plugins', 'marketplaces', sanitizedMarketplace)
```

### Lines Changed
- Lines 244–248 (added `sanitizedMarketplace` variable, updated `installPath` construction)

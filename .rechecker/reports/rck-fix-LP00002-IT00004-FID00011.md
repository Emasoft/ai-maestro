# Fix Report: FID00011

**File:** `components/plugin-builder/SkillPicker.tsx`
**Function:** `getSkillKey`
**Line:** 277

## Change Applied

In the `repo` case of `getSkillKey`, `skill.ref` was used directly without normalization. If `skill.ref` is empty or undefined, the key would contain an empty string segment, making it differ from a key generated with an explicit `'main'` ref — causing duplicate or mismatched skill entries.

**Before:**
```typescript
return `repo:${skill.url}:${skill.ref}:${skill.skillPath}`
```

**After:**
```typescript
return `repo:${skill.url}:${skill.ref || 'main'}:${skill.skillPath}`
```

## Root Cause

`skill.ref` is optional in the `PluginSkillSelection` type for the `repo` case. Without normalization, an undefined or empty ref produces a key like `repo:https://example.com/repo::path/to/skill` instead of `repo:https://example.com/repo:main:path/to/skill`, breaking key uniqueness and lookup consistency.

## Scope Check

The `getSkillKey` function has only one `repo` case switch arm. No other occurrences of the same pattern exist in the file. Fix is complete and confined to the single affected line.

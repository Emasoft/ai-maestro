# Fix Report: FID00017

**File:** `scripts/remote-install.sh`
**Function:** `act3_clone_and_build` (UPDATE path)
**Line:** 962

## Change Applied

Replaced the permissive `yarn install || maestro_warn` pattern (which allowed a broken update to proceed with missing dependencies) with a fail-fast pattern that halts execution immediately on failure.

**Before:**
```bash
yarn install || maestro_warn "yarn install had errors — continuing"
```

**After:**
```bash
yarn install || { maestro_fail "Failed to install dependencies during update."; exit 1; }
```

## Scope Check

Searched the entire file for other occurrences of `yarn install.*maestro_warn` — only one instance existed, which has been fixed.

## Result

DONE. The UPDATE path in `act3_clone_and_build` now exits immediately with a meaningful error message if `yarn install` fails, preventing any subsequent steps from running with an incomplete dependency set.

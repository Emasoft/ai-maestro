### [MF-001] SSRF check incomplete -- octal/hex/decimal IP representations bypass the private-IP blocklist
**File:** `app/api/hosts/health/route.ts`:24-33
**Original IDs:** CC-P6-A5-001
**Phases:** CC
The SSRF protection blocklist checks for literal strings like `127.0.0.1`, `10.x`, `192.168.x`, etc. However, a caller can bypass this using alternative IP representations that Node.js `fetch` resolves identically: octal (`0177.0.0.1`), decimal (`2130706433`), IPv4-mapped IPv6 (`::ffff:127.0.0.1`), hex (`0x7f000001`), and DNS rebinding. The URL is then passed to `checkRemoteHealth`, which makes HTTP requests to the target, enabling SSRF.

**Fix:** Resolve the hostname to an IP address before checking (e.g., using `dns.lookup`), then check the resolved IP. Alternatively, use an allowlist approach (only allow hosts in `hosts.json`).

## services/plugin-builder-service.ts (FID00012)

# Fix Report: FID00012 — plugin-builder-service.ts

**File fixed**: `services/plugin-builder-service.ts`
**Date**: 2026-03-22

## Fixes Applied

### Fix 1: `buildPlugin` catch block — `activeOps` decrement guarded with `didIncrement`

**Location**: `buildPlugin`, catch block (line ~439)

**Before**:
```typescript
  } catch (error) {
    // Plain decrement: always paired with the increment above, no Math.max needed.
    activeOps--
    console.error('Error starting plugin build:', error)
    return { error: 'Failed to start plugin build', status: 500 }
  }
```

**After**:
```typescript
  } catch (error) {
    // Only decrement if we actually incremented — if the mutex guard rejected
    // (didIncrement === false) we must not touch activeOps or the counter goes negative.
    if (didIncrement) { activeOps-- }
    console.error('Error starting plugin build:', error)
    return { error: 'Failed to start plugin build', status: 500 }
  }
```

**Root cause**: When `MAX_CONCURRENT_OPS` is reached, the mutex sets `didIncrement = false` and the function returns early with 429. If an exception then somehow fires in the catch, the unconditional `activeOps--` would drive the counter negative, allowing more concurrent operations than intended on recovery.

### Fix 4: `validateBuildConfig` — explicit `.` and `..` checks for marketplace/plugin names

**Location**: `validateBuildConfig`, marketplace skill validation block (lines ~178-195)

**Before**:
```typescript
      if (!SAFE_PATH_SEGMENT_RE.test(skill.marketplace)) {
        return `Marketplace skill "${skill.name}": Marketplace name contains invalid characters ...`
      }
      ...
      if (!SAFE_PATH_SEGMENT_RE.test(skill.plugin)) {
        return `Marketplace skill "${skill.name}": Plugin name contains invalid characters ...`
      }
```

**After** (explicit checks inserted BEFORE the regex tests):
```typescript
      if (skill.marketplace === '.' || skill.marketplace === '..') {
        return `Marketplace skill "${skill.name}": Marketplace name must not be '.' or '..'`
      }
      if (!SAFE_PATH_SEGMENT_RE.test(skill.marketplace)) { ... }
      ...
      if (skill.plugin === '.' || skill.plugin === '..') {
        return `Marketplace skill "${skill.name}": Plugin name must not be '.' or '..'`
      }
      if (!SAFE_PATH_SEGMENT_RE.test(skill.plugin)) { ... }
```

**Root cause**: `SAFE_PATH_SEGMENT_RE` is `/^[a-zA-Z0-9._-]+$/` which allows dots. The bare strings `.` and `..` both pass this regex. When passed to `path.join(os.homedir(), '.claude', 'plugins', 'marketplaces', skill.marketplace, skill.plugin, ...)`, a `.` is a no-op segment and `..` traverses up one directory — escaping the intended `marketplaces/` tree. The fix adds exact-equality guards before the regex so these values are rejected definitively.

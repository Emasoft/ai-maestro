## BUG 3 — agent.alias usage in display (FIXED)

**Root cause**: `agent.alias` is explicitly `@deprecated` in `types/agent.ts` ("Use 'name' instead. Kept temporarily for migration."). The display logic used `agent.label || agent.alias` in 4 places (condition check + text display, in both `normal` and `compact` variants), meaning agents whose `label` was unset would silently show nothing if `alias` was also absent — and agents with only `alias` set would display the deprecated field.

**Fix**: All four occurrences of `agent.label || agent.alias` replaced with `agent.label || agent.name`, using the canonical identity field as fallback. Comments added explaining the deprecation. The `mt-1`/`mt-3` spacing expression in the compact variant was updated to match.

---

## BUG 4 — getAvatarUrl empty agentId (FIXED)

**Root cause**: An empty string `agentId` causes the hash loop to not execute, leaving `hash = 0`, so `Math.abs(0) % 100 = 0` and every empty-id agent gets `/avatars/men_00.jpg` — indistinguishable from each other and from the first legitimately hashed agent.

**Fix**: Added an early return guard `if (!agentId) return '/avatars/men_00.jpg'` at the top of `getAvatarUrl`. This makes the fallback explicit and self-documenting rather than an accidental hash collision.

---

## BUG 5 — isEmoji regex matches single digits (FIXED)

**Root cause**: `\p{Extended_Pictographic}` in the Unicode standard includes the digit characters 0-9 (as part of the keycap sequence base characters), so passing a single digit like `"1"` could incorrectly return `true` from `isEmoji`, causing an agent whose `avatar` field contained a digit to render an emoji `<span>` instead of an image.

**Fix**: Added an explicit guard `if (str.length === 1 && /\d/.test(str)) return false` before the regex test. Keycap emoji sequences like "1️⃣" are always multi-character (digit + variation selector + combining enclosing keycap), so a single bare digit is never a renderable emoji. Comment updated to explain both the why and the exemption.

## Fix 2: `programArgs` empty-string check in `AgentCreationHelper` `onComplete` handler (REAL BUG — FIXED)

**Location**: Line 1449, `AgentCreationHelper` `onComplete` handler inside `AgentList`.

**Root cause**: `if (config.programArgs)` is truthy for any non-empty string, but an empty string `""` is already falsy in JS so that case is handled. **Change**:


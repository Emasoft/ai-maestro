   ```tsx
   onWake={isHibernated ? () => setWakeDialogAgent(agent) : undefined}
   ```
   This calls `setWakeDialogAgent` directly (not `handleWake`) ‚Äî it does not need event forwarding because it only opens a dialog and performs no navigation. Left unchanged per instructions.

3. **`onHibernate` prop before fix** (line 1042):
   ```tsx
   onHibernate={isOnline ? () => handleHibernate(agent) : undefined}
   ```
   Confirmed the bug: event was not passed, `stopPropagation` never fired.

// Event is optional to support programmatic calls that have no DOM event.

The hibernate button in the normal agent list view now correctly calls `e.stopPropagation()` via the forwarded event, preventing the parent agent-select click handler from firing simultaneously.

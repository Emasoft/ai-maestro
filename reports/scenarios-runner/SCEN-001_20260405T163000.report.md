## Issues Noticed (Non-Blocking)

### ISSUE-001: Role Plugin shows "ai-maestro-orchestrator-agent / custom" for new AUTONOMOUS agent
- **Noticed at:** Step S011
- **Severity:** INFO
- **Description:** Newly created AUTONOMOUS agent with "No plugin" selected in wizard shows "ai-maestro-orchestrator-agent / custom" in the Role Plugin banner at the top of the Overview tab. Expected: "No Role Plugin" or blank.
- **Suggested fix:** Investigate why the wizard sets a default role-plugin even when none is selected. May be a display-only issue.

### ISSUE-002: Delete button in sidebar doesn't trigger confirmation dialog via CDP
- **Noticed at:** Step S031
- **Severity:** WARN
- **Description:** The "Delete agent" button in sidebar hover actions (uid=29_3) does not produce a visible confirmation dialog when clicked via CDP. Multiple clicks had no effect. The button likely uses `window.confirm()` which CDP auto-dismisses. Had to use `evaluate_script` to call the DELETE API directly.
- **Suggested fix:** Replace `window.confirm()` with a custom React modal for the delete confirmation, consistent with other dialogs in the app (title assignment, password). This would make it compatible with CDP automation.

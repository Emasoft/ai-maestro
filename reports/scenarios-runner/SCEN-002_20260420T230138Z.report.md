## Bugs Found

### BUG-001: "Also delete agent folder" checkbox leaves residual `.claude/` subdirectory

- **Discovered at:** Post-S058 verification (after deleting alpha, beta, Zaire, scen002-manager with "Also delete agent folder" checkbox CHECKED)
- **Symptom:** `~/agents/<deleted-agent>/` folder is mostly removed, but a `.claude/` subdirectory (containing `settings.local.json` and possibly `amama/` memory dir) persists after delete.
- **Root cause (probable):** Race condition between `rm -rf ~/agents/<name>` (in `services/element-management-service.ts:5086`) and the plugin install/uninstall subsystem writing to `~/agents/<name>/.claude/settings.local.json` during the DeleteAgent pipeline. The final rm happens in G09, but plugin teardown hooks (ChangePlugin writing `settings.local.json`) may execute AFTER the rm on the same agent.
- **Status:** NOT FIXED in this run. Documented in 6+ prior runs (SCEN-020 MEMORY: "Recurring bug (6th consecutive run)"). Requires deeper investigation of plugin install/uninstall ordering vs folder deletion.
- **Workaround applied:** Moved leftover `~/agents/scen-test-*/` folders to `/tmp/scen002-leftover-*/` for audit (Rule 0 safe move). Cleanup path successfully restored registry state.

### ISSUE-003: Two consecutive password modals for title change (governance + sudo)

- **Noticed at:** S029/S035/S048
- **Severity:** INFO
- **Description:** Changing an agent's title triggers TWO modals in sequence:
  1. "Enter Governance Password" (Title Assignment Dialog's inline password confirm)
  2. "Confirm with password" (sudo-mode modal for PATCH /api/agents/[id]/title strict route)
- Both require the SAME governance password. The user types the password twice. UX is confusing (why two modals if same password?).
- **Suggested fix:** Combine the inline password confirm + sudo-token exchange into one modal, OR make the first modal pass the password directly to the sudo endpoint bypassing the second prompt.

### ISSUE-004: Help panel heading "AI Maestro Help" persists in DOM even when collapsed

- **Noticed at:** S007 debug
- **Severity:** INFO (minor accessibility concern)
- **Description:** The Help panel (right-side overlay) uses `translate-x-full pointer-events-none` to hide itself, but the `h2` heading "AI Maestro Help" remains in the DOM (just translated off-screen). This can confuse automated tools that search for "visible" elements via `offsetParent`/`getBoundingClientRect` (both return truthy for translated elements).
- **Suggested fix:** Consider `aria-hidden="true"` when collapsed, or conditionally render (mount/unmount).

### ISSUE-005: "Create new agent" + button is a 2-click toggle dropdown (not a direct action)

- **Noticed at:** S007
- **Severity:** INFO
- **Description:** Clicking the + button in sidebar header opens a dropdown with one item (Create Agent). Second click closes it. This is a 2-click flow for a single user intent. Since there's only one dropdown item, consider making the + button directly open the wizard.
- **Suggested fix:** If dropdown always has only "Create Agent" item, inline it. If other actions may be added later (e.g., "Import agent", "Create from template"), keep dropdown but make first click a tap-and-release (not click-to-toggle).

## Bugs Found & Fixed

### BUG-017: Team delete dialog missing governance password field
- **Discovered at:** Step S049b (Phase 7)
- **Symptom:** "Keep Agents" and "Delete Agents Too" buttons in the two-phase delete dialog did nothing. The DELETE API returned 400: "Team deletion requires governance password".
- **Root cause:** `handleDelete()` in `app/teams/page.tsx:86` sent `{ deleteAgents }` but the API requires `{ password, deleteAgents }`. The second phase of the delete dialog had no password input field.
- **Fix:** Added `deletePassword` and `deleteError` state variables, password input field to the agents phase of the delete dialog, error display, disabled buttons when no password entered, and password clearing on cancel. File: `app/teams/page.tsx`.
- **Verified at:** Steps S049b-S049c (retry after fix + rebuild)

## Issues Noticed (Non-Blocking)

### ISSUE-003: Sidebar delete button uses window.confirm() — CDP auto-dismisses
- **Noticed at:** Steps S066-S068 (cleanup)
- **Severity:** WARN
- **Description:** The sidebar agent delete button triggers `window.confirm()` which CDP auto-dismisses (returns false), preventing deletion through normal UI flow. This is the same issue documented in SCEN-001 as ISSUE-002.
- **Workaround:** Used `evaluate_script` to call DELETE API through the page's fetch.
- **Suggested fix:** Replace `window.confirm()` with a React modal dialog (same pattern as team delete).

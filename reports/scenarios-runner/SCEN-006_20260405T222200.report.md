## Bugs Found & Fixed

### BUG-018: Auto-discovery creates duplicate orphan agents (NOT FIXED)
- **Discovered at:** S017-S020
- **Symptom:** When wizard creates an agent, auto-discovery creates a second agent with UUID-based name from the tmux session
- **Root cause:** Wizard creates tmux session, auto-discovery scans tmux and registers the session as a new orphan agent because session name doesn't match registry entry lookup
- **Impact:** Title changes and profile views may target the wrong agent (the orphan instead of the wizard-created one)
- **Workaround:** Manually delete orphan agents after creation
- **Suggested fix:** Auto-discovery should check if a tmux session name matches any agent's tmux session name OR agent ID before creating an orphan entry

### BUG-019: Wizard silently fails on 400 error (NOT FIXED)
- **Discovered at:** S039
- **Symptom:** Clicking "Create Agent!" on the wizard summary does nothing when the server returns 400. No error message displayed to the user.
- **Root cause:** The wizard's handleCreate function catches the error but doesn't display it in the UI — the wizard stays on the summary step
- **Impact:** User has no feedback about why creation failed
- **Suggested fix:** Show a toast/banner with the server's error message when creation returns non-200

### BUG-020: No option to delete agent folder during agent deletion (FIXED)
- **Discovered at:** Phase CLEANUP
- **Symptom:** Deleting an agent leaves its working directory folder behind, blocking future agents with the same name
- **Root cause:** DELETE /api/agents/{id} only removed registry entry and tmux session, not the filesystem folder
- **Fix:** Added `?deleteFolder=true` query param to DELETE API and checkbox in DeleteAgentDialog
- **Files modified:** `DeleteAgentDialog.tsx`, `agents-core-service.ts`, `[id]/route.ts`, `page.tsx`
- **Safety:** Only deletes folders under `~/agents/` (hardcoded safety boundary)
- **Verified at:** Phase CLEANUP (folders deleted via restored state-wipe + manual cleanup)

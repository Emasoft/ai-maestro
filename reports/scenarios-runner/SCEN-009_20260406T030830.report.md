## Bugs Found & Fixed

### BUG-021: Prompt Builder textarea ignores CDP `fill` (React state not synced)
- **Discovered at:** Step S014
- **Symptom:** CDP `fill` tool sets the textarea value but React state doesn't update — character count shows 0, Send button remains disabled
- **Root cause:** React's `onChange` synthetic event handler doesn't fire when value is set programmatically via native setter + `dispatchEvent('input')`. The CDP automation and React's internal value tracking get out of sync.
- **Fix:** `components/TerminalView.tsx:115-121` — Added a native `input` event listener via `useEffect` on the prompt textarea ref that syncs `setPromptDraft(textarea.value)` on any input event, including programmatically dispatched ones.
- **Verified at:** After rebuild and reload, the `fill` tool correctly updates React state.

## Issues Noticed (Non-Blocking)

### ISSUE-004: MANAGER assigned COS title to ARCHITECT agent
- **Noticed at:** Step S021
- **Severity:** INFO
- **Description:** The MANAGER gave `swift-architect` the title `chief-of-staff` instead of `architect`. This is because the MANAGER replaced the auto-generated COS with the architect agent. The governance system allows this (COS is a valid team title), but the scenario expected an ARCHITECT title.
- **Suggested fix:** The MANAGER persona could be guided to keep the auto-COS and assign ARCHITECT to a separate agent. However, the MANAGER's decision was governance-compliant, so this is a persona behavior preference, not a bug.

## Summary of Concerns Requiring Attention

### Primary (SCEN-009)
- **Autonomous agent creation with non-deterministic compliance**: MANAGER agent creates 5+ sub-agents; compliance with prefix/workdir rules depends on the MANAGER agent's obedience to the prompt, not a hard gate. Mitigation options: (a) add a pre-creation assertion that the MANAGER's created agents all match `^scen9-` before proceeding, (b) add hard bail-out if any created agent's `workingDirectory` does not start with `/Users/emanuelesabetta/agents/scen9-`.
- **Broad substring cleanup in S028**: `"agents containing 'jsonl', 'swift', 'scen-mgr'"`. Change to strict `^scen9-` prefix.
- **Inconsistent prefix policy**: MANAGER is `scen-mgr-jsonl` but sub-agents are `scen9-*`. Unify to `scen9-mgr` for the MANAGER.

### Secondary (SCEN-007, 008)
- **`cos-` substring in cleanup**: Not currently hittable (no user agent starts with `cos-`), but brittle. Tighten to `^cos-scen7-` / `^cos-scen8-` prefix match in cleanup scripts.

### Tertiary (SCEN-012)
- **Bash-edits to `settings.local.json`** (S022, S026, S029) — Rule 6 concern, NOT a data-destruction concern. Files are inside the scenario's own agent folder. Not in scope for this audit but worth flagging.

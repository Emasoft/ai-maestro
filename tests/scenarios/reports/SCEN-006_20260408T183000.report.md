## Issues Noticed (Non-Blocking)

### ISSUE-001: MANAGER auth for wake command
- **Noticed at:** S014
- **Severity:** WARN
- **Description:** The MANAGER agent had difficulty authenticating to the wake API. It tried multiple approaches before succeeding with cookie-based auth via `/api/auth/login`. The `aimaestro-agent.sh wake` command didn't have a `--password` flag, forcing the agent to use curl directly.
- **Suggested fix:** Add `--password` flag to `aimaestro-agent.sh wake` command, or auto-inject an auth token for the MANAGER agent's session.

### ISSUE-002: Cannot delete offline/hibernated agents via sidebar
- **Noticed at:** S029, S031
- **Severity:** WARN
- **Description:** The sidebar "Delete agent" context button does not open the DeleteAgentDialog for agents that are offline or hibernated. The profile panel's Danger Zone (the only path to the dialog) requires an active terminal session. Offline agents are invisible in Zoom view. This means offline agents can only be removed by restoring registry backups.
- **Suggested fix:** Make the sidebar "Delete agent" button open the DeleteAgentDialog directly (as a modal over the current view) regardless of agent session state.

### ISSUE-003: Agent folder not deleted despite checkbox
- **Noticed at:** S030
- **Severity:** INFO
- **Description:** After deleting `scen006-codex-member` via the Danger Zone with "Also delete agent folder" checked, the folder `~/agents/scen006-codex-member/` still exists on disk. The deletion may have failed silently or the folder cleanup is async.
- **Suggested fix:** Verify folder deletion in the DELETE API response and show an error if it fails. Also check file permissions.
